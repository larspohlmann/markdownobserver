# Plan: Replace Session Auth with OAuth 2.0 + PKCE

> Generated by Claude Code from security audit findings in `reviews/security-audit.md`

## Context

`SessionTokenManager` stores tokens in `UserDefaults` (unencrypted). Moving to OAuth 2.0 + PKCE with Keychain storage resolves this and removes 320 lines of legacy auth code.

**Affected files:** `SessionTokenManager.swift`, `LegacyAuthMiddleware.swift`, `UserStore.swift`, `APIRouter.swift`

## Steps

### 1. Define auth protocol

```swift
protocol AuthenticationService: Sendable {
    func authenticate() async throws -> UserCredential
    func refresh(_ token: UserCredential) async throws -> UserCredential
    func revoke(_ token: UserCredential) async throws
}
```

### 2. Implement PKCE flow

- [x] Generate code verifier (43-128 chars, unreserved URI)
- [x] Derive code challenge via SHA256 + base64url
- [x] Open `ASWebAuthenticationSession` with challenge
- [x] Exchange auth code for tokens at `/oauth/token`
- [ ] Store refresh token in Keychain

### 3. Migrate `UserStore`

- [x] Replace `SessionTokenManager` with `AuthenticationService`
- [x] Add `AuthInterceptor` for Bearer token injection
- [ ] Handle concurrent refresh — queue during refresh
- [ ] Retry on 401: refresh token → retry request

## Risks

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Token rotation race | High | `actor` isolation on storage |
| Sessions invalidated | Medium | Parallel auth for 2 releases |
